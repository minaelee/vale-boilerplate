---
title: The OpenAPI generator
description: Use Fern's OpenAPI generator to convert your API defined using the Fern Definition format to an OpenAPI 3.1 specification file.
---

To export your API definition to the OpenAPI specification format from Fern Definition (Fern Def) files, you can use the `fern-openapi` generator. The OpenAPI generator outputs an OpenAPI 3.1 document for your API, which can be used with any OpenAPI tools. 

Here is an example of how you can configure the `fern-openapi` generator in your `generators.yml` file:

<CodeBlock title="generators.yml">
```yaml
- name: fernapi/fern-openapi
  version: 0.0.30
  config:
    format: yaml #options are yaml or json
  output:
    location: local-file-system
    path: ../openapi
  # github:
  #   repository: your-organization/your-repository
```
</CodeBlock>

## Configuration options 

The following options are available:

### Format

- `format`: Allows you to control the name of the generated namespace export and client class. By default, the exported namespace and client are named based on the organization and API names in the Fern Def.

**Type**: string, enum of 'json' | 'yaml'

**Default:** `yaml`

When configured, the generator outputs OAS files in the specified format.

#### Custom overrides

**Type:** object

**Default:** {}

When configured, the object is merged into the generated OAS file. This allows you to add custom fields to the specification.
